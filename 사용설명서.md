# 네이버 쇼핑 광고 수집기 — 사용설명서

## 이 프로젝트가 하는 일

네이버 쇼핑에서 특정 키워드를 검색했을 때 노출되는 **광고 중인 상품(브랜드스토어 제외)**을 자동으로 모아서, 웹 대시보드에서 확인할 수 있게 해주는 도구입니다.

구성 요소:

- `frontend/` — 결과를 확인하는 웹 대시보드 (React, Vercel에 배포됨)
- `crawler/` — 실제로 네이버 쇼핑을 방문해서 데이터를 모으는 프로그램 (Node.js + Playwright, **로컬 컴퓨터에서만 실행**)
- Supabase — 키워드 목록 / 브랜드 목록 / 수집 결과를 저장하는 데이터베이스

## 전체 흐름

1. 대시보드(키워드 관리)에서 수집하고 싶은 키워드를 등록
2. 내 컴퓨터에서 크롤러를 실행 (대시보드 버튼 또는 터미널 명령)
3. 크롤러가 네이버 쇼핑을 돌면서 광고 상품을 모아 Supabase에 저장
4. 대시보드(대시보드/수집 결과/브랜드 관리)에서 결과 확인

크롤러는 왜 서버에서 자동으로 안 돌고 꼭 내 컴퓨터에서 실행해야 하는지: 네이버가 자동화된 접근을 적극적으로 차단해서, 실제 로그인한 사람이 쓰는 진짜 브라우저 + 가끔 뜨는 캡차를 사람이 직접 풀어주는 방식으로만 안정적으로 동작합니다. 자세한 배경은 [crawler/README.md](crawler/README.md) 참고.

## 최초 1회 설정

### 1) 크롤러 설정

```bash
cd crawler
npm install
```

`crawler/.env.example`을 복사해 `crawler/.env`를 만들고 값을 채웁니다 (`frontend/.env`에 있는 값과 동일한 값을 넣으면 됩니다):

```
SUPABASE_URL=...
SUPABASE_ANON_KEY=...
```

### 2) 네이버 로그인 (최초 1회, 이후엔 세션 유지)

```bash
npm run chrome
```

전용 Chrome 창이 뜨면 네이버에 로그인합니다. 로그인 정보는 전용 프로필에 저장되어 다음 실행부터는 다시 로그인할 필요가 없습니다. **이 창을 끄면 안 됩니다** — 크롤러가 이 창을 통해 네이버에 접속합니다.

## 평소 사용법

### 방법 A — 대시보드 버튼으로 수집 (권장)

1. `crawler`에서 `npm run chrome` (Chrome이 안 떠있으면)
2. `crawler`에서 `npm run server` (로컬 수집 서버 실행, 켜둔 채로 둠)
3. `frontend`에서 `npm run dev`로 대시보드를 **로컬로** 띄우고 접속 (보통 `http://localhost:5173`)
4. 대시보드 우측 상단 **"수집 시작"** 버튼 클릭

> ⚠️ **Vercel에 배포된 사이트(`https://....vercel.app`)에서는 이 버튼이 동작하지 않습니다.** 배포된 사이트는 남이 실행한 내 컴퓨터의 프로그램을 조작할 방법이 없기 때문에, 결과를 "조회"하는 용도로만 씁니다. 수집을 시작하려면 위처럼 프론트엔드를 로컬로 띄워서 접속해야 합니다.

### 방법 B — 터미널에서 직접 수집

```bash
cd crawler
npm run chrome   # 최초 1회(또는 로그인 만료 시) 로그인
npm run dev      # 활성화된 모든 키워드를 한 번 수집하고 종료
```

## 캡차/차단이 뜨면

네이버가 가끔 사람인지 확인하는 캡차나 접속 제한 화면을 띄웁니다. 터미널에 `[BLOCKED]` 메시지가 뜨면:

1. `npm run chrome`으로 띄워둔 Chrome 창을 확인
2. 캡차가 보이면 직접 풀기
3. 터미널로 돌아와서 Enter 키 입력 → 자동으로 재시도

만약 "쇼핑 서비스 접속이 일시적으로 제한되었습니다" 같은 안내 화면이 뜬다면 캡차가 아니라 IP/세션이 일시적으로 막힌 상태입니다. 이건 사람이 풀 수 있는 게 아니라 시간이 지나야 풀리므로, 몇 시간 정도 기다렸다가 재시도해주세요. 계속 재시도하면 오히려 차단이 길어질 수 있습니다.

## 대시보드 화면 안내

| 메뉴 | 설명 |
|---|---|
| 📊 대시보드 | 전체 키워드/브랜드 수, 오늘 수집 건수, 최근 수집 시각, "수집 시작" 버튼 |
| 🔍 키워드 관리 | 수집할 키워드 등록/삭제/활성화 토글. 활성화된 키워드만 크롤러가 수집합니다 |
| 📦 수집 결과 | 지금까지 수집된 광고 상품 목록. 키워드/브랜드/상품명으로 검색 가능 |
| 🏷 브랜드 관리 | 수집 중 발견된 브랜드(판매처) 목록 |
| ⚙ 설정 | 아직 구현되지 않음 |

## 배포

- 프론트엔드는 Vercel에 배포되어 있고, `main` 브랜치에 push하면 자동으로 재배포됩니다.
- 크롤러는 배포되지 않는 로컬 전용 프로그램입니다.

## 문제 해결

| 증상 | 원인 / 해결 |
|---|---|
| 대시보드에 "로컬 수집 서버에 연결할 수 없습니다" 표시 | `crawler`에서 `npm run server`를 실행했는지 확인 |
| 수집이 바로 CDP 연결 에러로 실패 | `npm run chrome`으로 Chrome을 먼저 띄웠는지 확인 |
| 캡차가 자꾸 뜸 | 요청이 너무 잦은 상태일 수 있음. 키워드 수/페이지 수를 줄이거나 시간을 두고 재시도 |
| "쇼핑 서비스 접속이 일시적으로 제한되었습니다" 화면 | IP/세션이 일시적으로 차단된 상태. 몇 시간 기다렸다가 재시도 (재시도를 반복하면 더 길어질 수 있음) |
| 배포된 사이트에서 "수집 시작" 버튼이 안 먹음 | 의도된 동작. 로컬에서 `npm run dev`로 프론트를 띄워 접속해야 함 |
